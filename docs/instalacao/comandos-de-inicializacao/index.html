Comandos de Inicialização

Caso você já tenha realizado a etapa de instalação das dependências do projeto e realizado toda a etapa de configuração, é hora de executarmos a aplicação.

Server-side​

No diretório raiz do servidor /server, é necessário que executemos os seguintes comandos para inicializar a aplicação de forma correta:

yarn install​

O comando yarn install será responsável por realizar toda a instalação das dependências e pacotes pertinentes a aplicação, sendo assim, indispensável para o apropriado funcionamento da aplicação.

Exemplos //Imagens mostrando a execução do comando

yarn migrate​

O comando yarn migrate é responsável por criar as tabelas do banco de dados no projeto, deve ser utilizado na pasta raiz de /server.

yarn seed​

O comando yarn seed ira popular algumas tabelas com publicações, departamentos, usuários, chats, e outras tabelas da aplicação, com o proposito de testar novas funcionalidades da aplicação dentro do ambiente de desenvolvimento, sendo assim, se trata de um recurso que utiliza das propriedades do objeto de desenvolvimento para gerar todos recursos necessários.

yarn dev​

O comando yarn dev é utilizado inicializar o ambiente de desenvolvimento da aplicação, e ira utilizar as propriedades do objeto "development" para realizar inserções em tabelas do banco de dados ou requirir dados para armazéns de objetos que são responsáveis por armazenar arquivos estáticos.

yarn production​

O comando yarn production será responsável por inicializar o ambiente de produção, este, por sua vez, irá utilizar das propriedades do objeto "production" para realizar inserções em tabelas do banco de dados ou requirir dados para armazéns de objetos que são responsáveis por armazenar arquivos estáticos.

Client-side​

No diretório raiz do lado cliente da aplicação /app, é necessário que executemos os seguintes comandos para preparar o nosso projeto para desenvolvimento:

flutter pub get​

O comando flutter pub get se trata de um comando responsável por realizar a instalação de todos as dependências necessárias do front-end da aplicação.

flutter pub run​

O comando flutter pub run é um comando responsável por executar a aplicação front-end, através do uso do comando será disponibilizado algumas opções para executar o software em modo "debug", sendo estas opções: browser, nativo, android ou ios (caso seus respectivos emuladores estejam instalados dentro do ambiente local de desenvolvimento, conforme instruí o guia oficial de instalação do Flutter).
